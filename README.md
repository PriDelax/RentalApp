# Учёт аренды жилых помещений
Данное десктопное приложение разработано для автоматизации учёта аренды жилых помещений, контроля платежей и обслуживания инфраструктуры. Приложение предназначено для частных арендодателей, управляющих недвижимостью, и съёмщиков

## Технологии
* Язык программирования: Java
* Платформа: JavaFX
* База данных: PostgreSQL
* Подключение к БД: JDBC
* Архитектура: MVC + DAO
* Хэширование паролей:BCrypt
* Система сборки: Maven
* Среда разработки: IntelliJ IDEA Community Edition
* Операционная система: Windows 11
## Требования:
Перед запуском убедитесь, что установлены:
* Java Runtime Environment 17+ — скачать
* PostgreSQL 13+ — скачать
* Свободное место на диске: не менее 150 МБ
## Установка и запуск:
1. Настройка базы данных
Создайте базу данных в PostgreSQL:
```
CREATE DATABASE "RentalApp";
```
Настройте подключение в конфигурационном файле приложения(application.yml):
```
db.url=jdbc:postgresql://localhost:5432/RentalApp
db.username=postgres
db.password=ваш_пароль
```
## Использование
### Регистрация и авторизация
* При первом запуске нажмите «Регистрация»
* Выберите роль: Арендодатель или Съёмщик
* Заполните обязательные поля: логин, пароль, ФИО, телефон
* Для арендодателя дополнительно укажите платёжные реквизиты
* После регистрации войдите в систему
## Функции арендодателя
### Управление помещениями:
* Перейдите в раздел «Реестр помещений»
* Нажмите «Добавить помещение»
* Укажите адрес, тип, площадь, описание, статус
* Можно редактировать и удалять помещения (если нет активных договоров)
### Создание договора аренды:
* Перейдите в раздел «Договоры»
* Нажмите «Создать договор»
* Выберите съёмщика и свободное помещение
* Укажите даты начала/окончания, сумму аренды, график платежей
* Статус помещения автоматически изменится на «Занято»
### Добавление платежа:
* Перейдите в раздел «Платежи»
* Нажмите «Добавить платёж»
* Выберите договор, тип (аренда/залог/пени), сумму, дату, статус
* Виджеты на дашборде и календарь обновятся автоматически
### Управление заявками:
* Просмотрите список заявок от съёмщиков
* Выберите заявку со статусом «Новая»
* Нажмите «Назначить исполнителя» и выберите из списка
* После выполнения работ нажмите «Закрыть заявку»
### Финансовая аналитика:
* Просмотрите графики: доходы по месяцам, распределение по статусам, доходы по помещениям
* Используйте календарь платежей для планирования доходов
### Управление исполнителями:
* Добавьте исполнителей с указанием специальности, опыта, рейтинга
* Исполнители доступны для назначения на заявки
## Функции съёмщика
### Просмотр договоров:
* Отображаются все активные договоры съёмщика
* Можно просмотреть детали договора
### Просмотр платежей:
* Отображается история платежей с цветовой индикацией статусов
* Можно фильтровать по статусу (оплачено/не оплачено/частично)
* Создание заявки на ремонт:
* Нажмите «Создать заявку»
* Укажите описание проблемы, категорию (сантехника/электрика), приоритет
* Заявка отправляется арендодателю
### Отмена заявки:
Можно отменить заявку только со статусом «Новая»
### Профиль:
* Просмотр и редактирование личных данных (телефон, email)
* Смена пароля
* Поиск и фильтрация
* Во всех таблицах доступен поиск по ключевым полям
* Фильтрация по статусу, дате, типу и другим критериям
## Структура базы данных
Приложение использует следующие таблицы:
* tenants — съёмщики (id, login, password_hash, full_name, phone, email, passport, birth_date)
* landlords — арендодатели (id, login, password_hash, full_name, phone, email, passport, birth_date, payment_details)
* rooms — помещения (id, landlord_id, address, type, area, description, status)
* agreements — договоры (id, tenant_id, room_id, start_date, end_date, rent_amount, payment_schedule, status)
* payments — платежи (id, agreement_id, amount, payment_date, type, status)
* requests — заявки на ремонт (id, tenant_id, room_id, contractor_id, description, category, priority, status, created_at, closed_at)
* contractors — исполнители (id, full_name, phone, speciality, experience_years, rating, description)
## FAQ
Можно ли изменить роль пользователя после регистрации?
* Нет, роль выбирается при регистрации и не может быть изменена. Создайте новую учётную запись с нужной ролью.

Почему нельзя удалить помещение?
* Удаление запрещено, если с помещением связан активный договор. Сначала расторгните все договоры.

Почему нельзя удалить исполнителя?
* Удаление запрещено, если исполнитель назначен на активную заявку (статус «В работе»). Сначала отмените назначение.

Что делать, если приложение не запускается?
* Убедитесь, что установлен JRE 17+ и запущена служба PostgreSQL(pgAdmin4). Проверьте параметры подключения к базе данных в конфигурационном файле.
